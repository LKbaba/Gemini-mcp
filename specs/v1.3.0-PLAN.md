   ```typescript
   export function getModelConfig(modelId?: string): ModelConfig {
     if (!modelId) {
       return getDefaultModel();
     }

     // 检查是否为废弃模型
     if (modelId in DEPRECATED_MODEL_MAPPING) {
       const newModelId = DEPRECATED_MODEL_MAPPING[modelId];
       console.warn(
         `[Gemini MCP] Model "${modelId}" is deprecated and will be removed. ` +
         `Automatically using "${newModelId}" instead. ` +
         `Please update your configuration.`
       );
       return SUPPORTED_MODELS[newModelId] || getDefaultModel();
     }

     return SUPPORTED_MODELS[modelId] || getDefaultModel();
   }
   ```

# Migration Guide: v1.2.x → v1.3.0

## Overview

Version 1.3.0 updates the default Gemini model from `gemini-3-pro-preview` (retired 2026-03-09) to `gemini-3.1-pro-preview`.

## Do I Need to Do Anything?

**Short answer: No.**

This update is **fully backward compatible**. Old model names are automatically mapped to new models.

## What Changed?

### Default Model
- **Before**: `gemini-3-pro-preview`
- **After**: `gemini-3.1-pro-preview`

### Automatic Mapping
When you use the old model name, it's automatically mapped:
```json
{
  "model": "gemini-3-pro-preview"  // ← You specify this
}
// Internally becomes: "gemini-3.1-pro-preview"
```

You'll see a warning log:
```
[Gemini MCP] Model "gemini-3-pro-preview" is deprecated and will be removed.
Automatically using "gemini-3.1-pro-preview" instead.
Please update your configuration.
```

## Recommended Actions

### 1. Update Explicit Model References (Optional)

If your code explicitly specifies the model:

**Before**:
```typescript
{
  "name": "gemini_analyze_content",
  "arguments": {
    "filePath": "./src/index.ts",
    "model": "gemini-3-pro-preview"  // ← Old
  }
}
```

**After**:
```typescript
{
  "name": "gemini_analyze_content",
  "arguments": {
    "filePath": "./src/index.ts",
    "model": "gemini-3.1-pro-preview"  // ← New
  }
}
```

### 2. No Changes Needed for Default Usage

If you don't specify a model, you're already using the new default:

```typescript
{
  "name": "gemini_analyze_content",
  "arguments": {
    "filePath": "./src/index.ts"
    // No model specified → uses gemini-3.1-pro-preview
  }
}
```

### 3. Search Tool Unchanged

The search tool continues to use `gemini-3-flash-preview` by default (no changes needed).

## What's New in Gemini 3.1 Pro?

- **Agentic Reasoning**: Better multi-step planning
- **Performance**: ~15-20% faster responses
- **Reliability**: More stable output quality
- **Same Price**: $1.25/M input, $5.00/M output

## Troubleshooting

### Issue: "Model not found" error

**Cause**: You're using a very old version.

**Solution**: Update to v1.3.0:
```bash
npm install @lkbaba/mcp-server-gemini@latest
```

### Issue: Warning logs are annoying

**Cause**: You're still using the old model name.

**Solution**: Update your code to use `gemini-3.1-pro-preview`.

## Timeline

- **2026-03-09**: `gemini-3-pro-preview` retired by Google
- **2026-03-09**: v1.3.0 released with automatic mapping
- **2026-06-17**: `gemini-2.5-pro` will be retired (future update)

## Questions?

- GitHub Issues: https://github.com/LKbaba/Gemini-mcp/issues
- Documentation: https://github.com/LKbaba/Gemini-mcp#readme
